94 /*
95 
96 This image category implements image vectorization using Peter Selinger's potrace algorithm and OSS code.
97 
98 It works as follows:
99 
100 // stage 1:
101 
102 1. A 24-bit bit image is made from the NSImage contents (ensures that regardless of image format, we have a standard RGB bitmap to work from)
103 2. The image is analysed using a quantizer to determine the best set of colours needed to represent it at the chosen sampling value
104 3. A DKImageVectorRep is allocated for each colour. This allocates a bitmap data structure that potrace can work with.
105 4. The 24-bit image is scanned and the corresponding bits in the bit images are set according to the index value returned by the quantizer
106 5. Empty bitplanes are discarded
107 6. The resulting list of DKImageVectorRep objects is returned
108 
109 // stage 2:
110 
111 7. The client code requests the vector path from the DKImageVectorRep. This triggers a call to potrace with the generated bitmap for that colour
112 8. The client assembles the resulting paths into objects that can use the paths, for example DKDrawableShapes.
113 9. The client assembles the shapes into a group and adds it to the drawing.
114 
115 (Steps 8 and 9 are what is done in DrawKit - other client code might have other ideas).
116 
117 Note that the API to this operates at a high level in a category on DKImageShape - see DKImageShape+Vectorization.
118 
119 */
