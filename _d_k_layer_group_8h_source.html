95 /*
96 
97 A layer group is a layer which maintains a list of other layers. This permits layers to be organised hierarchically if
98 the application wishes to do so.
99 
100 DKDrawing is a subclass of this, so it inherits the ability to maintain a list of layers. However it doesn't honour
101 every possible feature of a layer group, particularly those the group inherits from DKLayer. This is because
102 DKLayerGroup is actually a refactoring of DKDrawing and backward compatibility with existing files is required. In particular one
103 should take care not to add a DKDrawing instance to a layer group belonging to another drawing (or create circular references).
104 
105 The stacking order of layers is arranged so that the top layer always has the index zero, and the bottom is at (count -1).
106 In general your code should minimise its exposure to the actual layer index, but the reason that layers are stacked this
107 way is so that a layer UI such as a NSTableView doesn't have to do anything special to view layers in a natural way, with
108 the top layer at the top of such a table. Prior to beta 3, layers were stacked the other way so such tables appeared to
109 be upside-down. This class automatically reverses the stacking order in an archive if it detects an older version.
110 
111 */
