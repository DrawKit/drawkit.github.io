235 /*
236 
237 This is a group objects that can group any number of shapes or paths.
238 
239 It inherits from DKDrawableShape so that it gets the usual sizing and rotation behaviours.
240 
241 This operates by establishing its own coordinate system in which the objects are embedded. An informal protocol is used that allows a shape or
242 path to obtain the transform of its "parent". When that parent is a group, the transform is manipulated such that the path is modified just
243 prior to rendering to allow for the group's size, rotation, etc.
244 
245 Be aware of one "gotcha" with this class - a bit of a chicken-and-egg situation. When objects are grouped, they are offset to be local to the group's
246 overall location. For grouping to be undoable, the objects being grouped need to have a valid container at the time this location offset is done,
247 so that there is an undo manager available to record that change. If not they might end up in the wrong place when undoing the "group" command.
248 
249 For the normal case of grouping existing objects within a layer, this is not an issue, but can be if you are programmatically creating groups.
250 
251 */
