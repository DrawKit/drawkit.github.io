34 /*
35 
36 This category provides high-level Boolean Operation commands for a drawing layer. It requires the
37 NSBezierPath+GPC category to function.
38 
39 The operations here are user-level ops, and can be simply hooked to menu commands. The operations proceed as follows:
40 
41 Union:
42 
43 Two or more objects in the selection are replaced by a single shape object whose path is the union of all the selected object's paths.
44 Path objects are converted to shape objects prior to forming the union. The result object does not preserve the rotation angle
45 of the original objects. The result inherits the style of the topmost object. The result is always a shape even if some or all
46 of the contributing objects are paths.
47 
48 Difference:
49 
50 Exactly two objects must be in the selection. The shape or path that is topmost is unchanged, but acts as a "cookie cutter"
51 for the other object, which is replaced by the result. The result inherits the style and type of the object it replaces.
52 Rotation angle is preserved for shapes.
53 
54 Intersection:
55 
56 Exactly two objects must be in the selection. Path objects are converted to shape objects before computing the intersection.
57 The original objects are replaced by the intersection unless the intersection is empty, in which case this is a no-op. The
58 result inherits the style of the topmost original object. Rotation angle is not preserved; the result is always a shape even if
59 one or both of the operands is a path object.
60 
61 Xor:
62 
63 As for intersection.
64 
65 Combine:
66 
67 Two or more objects must be in the selection. Appends each path to the lowest one using appendBezierPath. The result is like an Xor
68 or a union, depending on the disposition of the paths. The advantage of this is that no path flattening is required.
69 
70 Note that the design choices here about what type the result is, how stacking order affects the result, and so forth are intended
71 to provide the most natural and obvious outcomes for a typical drawing type program. That's why these operations must be considered
72 high level. If you want to implement some other behaviour built from boolean operations on paths, you have every freedom to do so,
73 since the code this calls operates purely at the abstract NSBezierPath level.
74 
75 */
