61 /*
62 
63  This uses a similar algorithm to DKBSPObjectStorage but instead of indexing the objects it stores them directly by retaining them in additional arrays
64  within the BSP tree. This is likely to be faster than the indexing approach though profiling is needed to confirm this.
65 
66  To facilitate correct z-ordering, each object stores its own Z-position and the objects are sorted on this property when necessary. Objects need to be
67  renumbered when indexes change.
68 
69  The trade-off here is that drawing speed should be faster but object insertion, deletion and changing of Z-position may be slower.
70 
71 */
