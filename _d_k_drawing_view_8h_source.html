144 /*
145 
146 DKDrawingView is the visible "front end" for the DKDrawing architecture.
147 
148 A drawing can have multiple views into the same drawing data model, each with independent scales, scroll positions and so forth, but
149 all showing the same drawing. Manipulating the drawing through any view updates all of the views. In many cases there will only be
150 one view. The views are not required to be in the same window.
151 
152 The actual contents of the drawing are all supplied by DKDrawing - all this does is call it to render its contents.
153 
154 If the drawing system is built by hand, the drawing owns the view controller(s), and some other object (a document for example) will own the
155 drawing. However, like NSTextView, if you don't build a system by hand, this creates a default one for you which it takes ownership
156 of. By default this consists of 3 layers - a grid layer, a guide layer and a standard object layer. You can change this however you like, it's
157 there just as a construction convenience.
158 
159 Note that because the controllers are owned by the drawing, there is no retain cycle even when the view owns the drawing. Views are owned by
160 their parent view or window, not by their controller.
161 
162 */
