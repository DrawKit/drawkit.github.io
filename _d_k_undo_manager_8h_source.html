58 /*
59 
60 This subclass of NSUndoManager can coalesce consecutive tasks that it receives so that only one task is recorded to undo a series of
61 otherwise identical ones. This is very useful when interactively editing objects where a large stream of identical tasks can be
62 received. It is largely safe to use with coalescing enabled even for normal undo situations, so coalescing is enabled by default.
63 
64 It also records a change count which is an easy way to check if the state of the undo stack has changed from some earlier time -
65 just compare the change count with one you recorded earlier.
66 
67 ************* NOTE - THIS DOES NOT WORK - DO NOT ENABLE GROUP DEFERRAL!! ***************
68 
69 Group deferral is another useful thing that works around an NSUndoManager bug. When beginUndoGrouping is called, the group is not
70 actually opened at that point - instead it is flagged as deferred. If an actual task is received, the group is opened if the
71 defer flag is set. This ensures that a group is only created when there is something to put in it - NSUndoManager creates a
72 bogus Undo item on the stack for empty groups. This allows client code to simply open a group on mouse down, do stuff in dragged,
73 and close the group at mouse up without creating bogus stack states.
