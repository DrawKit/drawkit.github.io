732 /*
733  A drawable object is owned by a DKObjectDrawingLayer, which is responsible for drawing it when required and handling
734  selections. This object is responsible for the visual representation of the selection as well as any content.
735 
736  It can draw whatever it likes within <bounds>, which it is responsible for calculating correctly.
737 
738  hitTest can return an integer to indicate which part was hit - a value of 0 means nothing hit. The returned value's meaning
739  is otherwise private to the class, but is returned in the mouse event methods.
740 
741  This is intended to be a semi-abstract class - it draws nothing itself. Subclasses include DKDrawableShape and DKDrawablePath -
742  often subclassing one of those will be more straightforward than subclassing this. A subclass must implement NSCoding and
743  NSCopying to be archivable, etc. There are also numerous informal protocols for geometry, snapping, hit testing, drawing and ungrouping
744  that need to be implemented correctly for a subclass to work fully correctly within DK.
745 
746  The user info is a dictionary attached to an object. It plays no part in the graphics system, but can be used by applications
747  to attach arbitrary data to any drawable object.
748 
749 */
