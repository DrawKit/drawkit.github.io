62 /*
63 
64 This is used to permit setting up KVO in a simpler manner than comes as standard.
65 
66 The idea is that each class simply publishes a list of the observable properties that an observer can observe. When the observer wants to
67 start observing all of these published properties, it calls setUpKVOForObserver: conversely, tearDownKVOForObserver: will stop the
68 observer watching all the published properties.
69 
70 Subclasses can also override these methods to be more selective about which properties are observed, or to propagate the message to
71 additional observable objects they own.
72 
73 This class also works around a bug or oversight in the KVO implementation (in 10.4 at least). When an array is changed, the old
74 value isn't sent to the observer. To allow this, we record the old value locally. An observer can then call us back to get this
75 old array if it needs to (for example, when building an Undo invocation).
76 
77 The undo relay class provides a standard implementation for using KVO to implement Undo when using GCObservables. The relay needs
78 to be added as an observer to any observable and given an undo manager. Then it will relay undoable actions from the observed
79 objects to the undo manager and vice versa, implementing undo for all keypaths declared by the observee.
