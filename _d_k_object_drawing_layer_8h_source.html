199 /*
200 
201 This layer adds the concept of selection to drawable objects as defined by DKObjectOwnerLayer. Selected objects are held in the -selection
202 list, which is a set (there is no order to selected objects per se - though sometimes the relative Z-stacking order of objects in the selection
203 is needed, and the method -selectedObjectsPreservingStackingOrder et. al. will provide that.
204 
205 Note that for selection, the locked state of owned objects is ignored (because it is OK to select a locked object, just not to
206 do anything with it except unlock it).
207 
208 Commands directed at this layer are usually meant to to go to "the selection", either multiple or single objects.
209 
210 This class provides no direct mouse handlers for actually changing the selection - typically the selection and other manipulation
211 of objects in this layer is done through the agency of tools and a DKToolController.
212 
213 The actual appearance of the selection is mainly down to the objects themselves, with some information supplied by the layer (for example
214 the layer's selectionColour). Also, the layer's (or more typically the drawing's) DKKnob class is generally used by objects to display their
215 selected state.
216 
217 */
