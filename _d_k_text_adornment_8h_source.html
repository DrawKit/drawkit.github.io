235 /*
236 
237 This renderer allows text to be an attribute of any object.
238 
239 This renderer also implements text-on-a-path. To do this, set the layoutMode to kDKTextLayoutAlongPath. Some attributes are ignored in
240 this mode such as angle and vertical alignment. However all textual attributes are honoured.
241 
242  Text adornments extensively cache information internally to speed drawing by avoiding recalculation of various things. The cache is a
243  dictionary which can store many different cached items. The cache is invalidated by changes arising in the client object and in the
244  state of internal data, and in addition the same cache is passed to text-on-path and other lower level methods which they use to avoid
245  similar lengthy recalculations. The caching is transparent to client objects but may need to be taken into account if subclassing or
246  using alternative helper objects, etc.
247 
248  The text content is stored and suplied by DKTextSubstitutor which is able to build strings by reading an object's metadata and combining it with
249  other fixed content. See that class for details.
250 
251 */
