80 /*
81 
82 DKDrawingTool is the semi-abstract base class for all types of drawing tool. The point of a tool is to act as a translator for basic mouse events and
83 convert those events into meaningful operations on the target layer or object(s). One tool can be set at a time (see DKToolController) and
84 establishes a "mode" of operation for handling mouse events.
85 
86 The tool also supplies a cursor for the view when that tool is selected.
87 
88 A tool typically targets a layer or the objects within it. The calling sequence to a tool is coordinated by the DKToolController, targeting
89 the current active layer. Tools can change the data content of the layer or not - for example a zoom zool would only change the scale of
90 a view, not change any data.
91 
92 Tools should be considered to be controllers, and sit between the view and the drawing data model.
93 
94 Note: do not confuse "tools" as DK defines them with a palette of buttons or other UI - an application might implement an interface to
95 select a tool in such a way, but the buttons are not tools. A button could store a tool as its representedObject however. These UI con-
96 siderations are outside the scope of DK itself.
97 
98 */
