109 /*
110 
111 DKImageShape is a drawable shape that displays an image. The image is scaled and rotated to the path bounds and clipped to the
112 path. The opacity of the image can be set, and whether the image is drawn before or after the normal path rendering.
113 
114 This object is quite flexible - by changing the path clipping and drawing styles, a very wide range of different effects are
115 possible. (n.b. if you don't attach a style, the path is not drawn at all [the default], but still clips the image. The default
116 path is a rect so that the entire image is drawn.
117 
118 There are two basic modes of operation - scaling and cropping. Scaling fills the shape's bounds with the image. Cropping keeps the image at its
119 original size and allows the path to clip it as it is resized. In both cases the image offset can be used to position the image within the bounds.
120 A hotspot is added to allow the user to drag the image offset position around.
121 
122  Image shapes automatically manage image data efficiently, such that if there is more than one shape with the same image, only one copy of
123  the data is maintained, and that data is the original compressed data from the file (if it did come from a file). This data sharing is
124  facilitated by a central DKImageDataManager object, which is managed by the drawing. Note that using certian operations, such as creating
125  the shape with an NSImage will bypass this benefit.
126 
127 */
