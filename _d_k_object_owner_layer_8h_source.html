797 /*
798 
799 This layer class can be the owner of any number of DKDrawableObjects. It implements the ability to contain and render
800 these objects.
801 
802 It does NOT support the concept of a selection, or of a list of selected objects (DKObjectDrawingLayer subclasses this to
803 provide that functionality).
804 
805 This split between the owner/renderer layer and selection allows a more fine-grained opportunity to subclass for different
806 application needs.
807 
808 Layer caching:
809 
810 When a layer is NOT active, it may boost drawing performance to cache the layer's contents offscreen. This is especially beneficial
811 if you are using many layers. By setting the cache option, you can control how caching is done. If set to "none", objects
812 are never drawn using a cache, but simply drawn in the usual way. If "pdf", the cache is an NSPDFImageRep, which stores the image
813 as a PDF and so draws it at full vector quality at all zoom scales. If "CGLayer", an offscreen CGLayer is used which gives the
814 fastest rendering but will show pixellation at higher zooms. If both pdf and CGLayer are set, both caches will be created and
815 the CGLayer one used when DKDrawing has its "low quality" hint set, and the PDF rep otherwise.
816 
817 The cache is only used for screen drawing.
818 
819 NOTE: PDF caching has been shown to be actually slower when there are many objects, espcially with advanced storage in use. This is
820 because it's an all-or-nothing rendering proposition which direct drawing of a layer's objects is not.
821 
822 */
