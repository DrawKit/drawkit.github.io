661 /*
662 
663 This is a useful container class that is like a "super dictionary" or maybe a "micro-database". As well as storing an object using a key,
664 it allows the object to be associated with none, one or more categories. An object can be a member of any number of categories.
665 
666 As objects are added and used, they are automatically tracked in a "recently added" and "recently used" list, which can be retreived at any time.
667 
668 As with a dictionary, an object is associated with a key. In addition to storing the object against that key, the key is added to the categories
669 that the object is a member of. This facilitates category-oriented lookups of objects.
670 
671 */

734 /*
735 The cat manager supports a UI based on menu(s). To assist, the DKCategoryManagerMenuInfo class is used to "own" a menu - the cat manager keeps a list of these.
736 
737 When the CM is asked for a menu, this helper object is used to create and manage it. As the CM is used (items and categories added/removed) the menu helpers are
738 informed of the changes and in turn update the menus to match by adding or deleting menu items. This is necessary because when the CM grows to a significant number
739 of items, rebuilding the menus is very time-consuming. This way performance is much better.
