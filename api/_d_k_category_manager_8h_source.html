596 /*
597 
598 This is a useful container class that is like a "super dictionary" or maybe a "micro-database". As well as storing an object using a key,
599 it allows the object to be associated with none, one or more categories. An object can be a member of any number of categories.
600 
601 As objects are added and used, they are automatically tracked in a "recently added" and "recently used" list, which can be retreived at any time.
602 
603 As with a dictionary, an object is associated with a key. In addition to storing the object against that key, the key is added to the categories
604 that the object is a member of. This facilitates category-oriented lookups of objects.
605 
606 */
