598 /*
599 
600 This is a useful container class that is like a "super dictionary" or maybe a "micro-database". As well as storing an object using a key,
601 it allows the object to be associated with none, one or more categories. An object can be a member of any number of categories.
602 
603 As objects are added and used, they are automatically tracked in a "recently added" and "recently used" list, which can be retreived at any time.
604 
605 As with a dictionary, an object is associated with a key. In addition to storing the object against that key, the key is added to the categories
606 that the object is a member of. This facilitates category-oriented lookups of objects.
607 
608 */
