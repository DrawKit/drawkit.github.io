129 /*
130 
131 This category on NSBezierPath converts to and from the gpc_polygon data structure used by
132 the wonderful gpc (general polygon clipping) lib. This lib is used to perform boolean ops on paths.
133 
134 Note that at present paths are flattened into polygons and so curve control points, etc are not preserved.
135 
136 The curve-fitting is accomplished using 3rd party code from Lib2Geom, which is in turn a C++ implementation
137 of the classic Graphics Gems code. Curve-fitting is controlled by the "simplifying policy" that you set. By
138 default it's set to 'auto', meaning that if either of the original paths contains curves, curve fitting will
139 be done on the result, but if both source paths only have line segments, it won't be. This preserves sharp-cornered
140 shapes such as rects, etc.
141 
142 For simplifying a path at any other time, you must pass a flattened path. Simplifying really means "unflattening".
