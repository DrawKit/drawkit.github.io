769 /*
770 
771 This layer adds the concept of selection to drawable objects as defined by DKObjectOwnerLayer. Selected objects are held in the -selection
772 list, which is a set (there is no order to selected objects per se - though sometimes the relative Z-stacking order of objects in the selection
773 is needed, and the method -selectedObjectsPreservingStackingOrder et. al. will provide that.
774 
775 Note that for selection, the locked state of owned objects is ignored (because it is OK to select a locked object, just not to
776 do anything with it except unlock it).
777 
778 Commands directed at this layer are usually meant to to go to "the selection", either multiple or single objects.
779 
780 This class provides no direct mouse handlers for actually changing the selection - typically the selection and other manipulation
781 of objects in this layer is done through the agency of tools and a DKToolController.
782 
783 The actual appearance of the selection is mainly down to the objects themselves, with some information supplied by the layer (for example
784 the layer's selectionColour). Also, the layer's (or more typically the drawing's) DKKnob class is generally used by objects to display their
785 selected state.
786 
787 */
