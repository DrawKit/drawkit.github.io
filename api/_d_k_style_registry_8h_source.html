372 /*
373 
374 The style registry is a singleton category manager instance that consolidates styles from a variety of sources into a single app-wide "database"
375 of styles, organised into categories.
376 
377 Styles can come from these sources:
378 
379 1. The application defaults, if the app is launched with no library preferences available (i.e. first run).
380 
381 2. The styles library, which is the complete registry saved to disk (user prefs) at quit time.
382 
383 3. A document, when it is opened.
384 
385 4. A separate file containing just styles.
386 
387 5. A new style being created and registered by the user as the app is used.
388 
389 -------------------------------------------------------------------------------------------------------------------------------------------------
390 The point of the registry is twofold:
391 
392 A. It permits the construction of a UI for accessing pre-built styles and applying them to objects in a drawing. By organising styles into categories,
393 potentially large number of styles may be managed effectively.
394 
395 B. It tracks styles across any number of documents as they are created. For example if a document uses a particular registered style, when that
396 document is reopened at a later date, the style can be recognised and linked to the same style in the registry. If the style has changed in the
397 meantime so that there is a difference between the saved style and the currently registered version, the user can be offered the option to
398 update the style in the document to match the registry, update the registry to match the document, or to re-register the style as a new version.
399 
400 Note the registry is not mandatory - an app using DK can use styles without registering them if it wishes. The advantage of the registry is that it
401 permits styles to persist and be tracked across multiple documents, saving the user from having to redefine styles for every new graphic.
402 
403 -------------------------------------------------------------------------------------------------------------------------------------------------
404 In order for the registry to uniquely and positively identify a style, its unique ID is used as its key. The unique ID is assigned when the style
405 first comes into existence and cannot be changed. It is a string representation of a UUID so is guaranteed unique.
406 
407 UUID's are not very user friendly and should never be exposed by an application's user interface. Thus a style also has an ordinary descriptive
408 name which can be displayed in the UI. Such names are not guaranteed to be unique however, as the user is free to enter whatever name they wish.
409 When a style is first registered the name may be changed to avoid a collision with an already registered style having the same name - this is
410 done by appending 1, 2, 3 etc until the name doesn't collide. However this is just done to disambiguate the style for the user - it is not intended
411 to guarantee uniqueness so that the name can be used as a key to the object.
412 
413 A user interface will want to use the ordinary names, but internally must be set up to use the unique ID or the object itself to avoid any confusion
414 as to which style is actually being referred to. For example a menu of available styles could use the UUID or the object as the represented object
415 of the menu item. DO NOT USE THE ORDINARY NAME AS A KEY.
416 
417 -------------------------------------------------------------------------------------------------------------------------------------------------
418 
419 Locking. Styles in the registry are usually locked. This is to prevent accidental alteration of a style that may be being used across many
420 documents. When a style is added to the registry, it should be unlocked (because the name might need to be automatically changed) but the
421 registry will lock the style upon a successful operation.
422 
423 Styles in a document. Styles used in a document may or may not be registered. A user may never register a style, but still have many styles
424 defined in a document. Such styles work normally and can be copied and pasted and shared between objects if set to do so - use of styles is not
425 dependent in any way upon the registry. When the document is saved, a flag as to whether the style was registered or not is saved with it. When
426 the document is opened again later, such flagged styles will be optionally reconnected to the style registry so that the actual style used is
427 always the registered style, so if the style is updated, existing documents are offered the opportunity to use the updated version.
428 
429 As well as any user-defined categories, documents may wish to create a temporary category using the document's name which gives the user a
430 way to quickly discover the complete set of registered styles used in a document. The category should be removed when the document is closed.
431 DKDrawDocument implements this behaviour by default, so if your document class is based on it, your app can get this feature for free.
432 
433 -------------------------------------------------------------------------------------------------------------------------------------------------
434 
435 Registering a style. The style registry performs the following steps:
436 
437 1. It checks that the style is not already registered. If it is, it does no more.
438 
439 2. It checks that the style is unlocked. If not, an exception is thrown.
440 
441 3. It resolves the style's name so that there are no collisions with any existing style's name, and changes the style's name if needed.
442 
443 4. It creates any new categories as requested (depends on the particular method used to register the style). If no specific categories are
444  requested, the style is added to the default category.
445 
446 5. It adds the style to the registry using its unique ID as the key.
447 
448 6. It locks the style.
449 
450 -------------------------------------------------------------------------------------------------------------------------------------------------
451 
452 Cut/Paste: cut and paste of styles works independently of the registry, including dealing with shared styles. See DKStyle for more info.
453 
454 */
