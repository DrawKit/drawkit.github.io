237 /*
238 
239 This class is a simple document type that owns a drawing instance. It can be used as the basis for any drawing-based
240 document, where there is a 1:1 relationship between the documnent, the drawing and the main drawing view.
241 
242 You can subclass to add functionality without having to rewrite the drawing ownership stuff.
243 
244 This also handles standard printing of the drawing
245 
246 Note that this is expected to be set up via the associated nib file. The outlet m_mainView should be set to the DKDrawingView in the window. Inherited
247 outlets such as window should be set as normal (File's Owner is of course, this object). If you forget to set the m_mainView outlet things won't work
248 properly because the document won't know which view to link to the drawing it creates. What will happen is that the unconnected view will work, and the first
249 time it goes to draw it will detect it has no back-end, and create one automatically. This is a feature, but in this case can be misleading, in that the drawing
250 you *see* is NOT the drawing that the document owns. The m_mainView outlet is the only way the document has to know about the view it's supposed to connect to
251 its drawing.
252 
253 If you subclass this to have more views, etc, bear this in mind - you have to consider how the document's drawing gets hooked up to the views you want. Outlets
254 like this are one easy way to do it, but not the only way.
255 
256 */
